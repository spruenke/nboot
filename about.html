About Resampling

Basic Idea

Sloppily said, Resampling is a nonparametric Maximum-Likelihood Approach. The idea is, to draw new (sub)samples from the original data and estimate statistics based on the (sub)samples and finally combine these estimates to a final one. So, we iterate nboot times and in each iteration we draw a (sub)sample, estimate the statistic of interest, save it and after all iterations use something like mean/median/etc. to fit them into one estimator. Furthermore, we mimic the empirical distribution and can use it for set-estimates (e.g. confidence regions).

Advantages and Disadvantages

The main advantage of resampling techniques is, that they offer a way to overcome small sample size problems. Since especially frequentist statistics rely on large sample properties and asymptotic behavior. On small samples, the statistics can become unreliable fast. Resampling can sometimes solve this problem.

However, there’s also a downside to resampling. If the small sample is a ‘’bad sample’’, resampling cannot solve the problem since it relies on the sample.

Methodology and Terminology

In the following you can find some formal ideas of the methods as well as a brief description of how the different resampling techniques work. Generally, we want to use the law of large numbers and the convergence of the empirical distribution to the actual distribution:

\[ \frac{1}{n}\sum_{i=1}^n X_i \quad \stackrel{\mathbb{P}-a.s.}{\to} \quad \mathbb{E}[X]\\ \Rightarrow F_n(x) \quad \stackrel{\mathbb{P}-a.s.}{\to} \quad F^X(x) \]

Nonparametric Bootstrap

As the name says, in the nonparametric setting we want to avoid any parametric assumptions about the distribution of the sample. This package supports two types of nonparametric bootstrap, the classical one (for one- and many-sample problems) as well as the groupwise approach (only many-sample problems).

Classical Nonparametric Bootstrap

This approach is quite simple. We fix our data \(X = \{X_{11}, ..., X_{ji}, ..., X_{kn_{k}}\}\) and sample randomly from this. In a many-sample problem we draw \(k \cdot n\) times from this sample and afterwards split into \(k\) groups. In each case, we draw with replacement to ensure equal probability for each observation.

Groupwise Nonparametric Bootstrap

The groupwise approach is similar to the many-sample version of the classical approach, with the only difference that we fix our data as groups before sampling \(X_j = \{X_{j1}, ..., X_{ji}, ..., X_{jn_j}\}\) and sample with replacement for each group separately.

Parametric Bootstrap

The parametric bootstrap makes an assumption about the distribution of the data. Here, the data itself is only fixed to estimate parameters for the distribution and then sample directly from that distribution.

Wild Bootstrap

The wild bootstrap is an approach where we fix our data \(X = \{X_{11}, ..., X_{ji}, ..., X_{kn_{k}}\}\), then center it, such that \(Z_{ji} = X_{ji} - \overline{X_{ji}}\). Now, we sample from a symmetric distribution with \(\mathbb{E}[\epsilon] = 0\) and create a bootstrap sample \(Z \odot \epsilon\). This can be done in a groupwise or non-groupwise approach.

Cross-Validation

k-fold Cross Validation is a resampling technique often used in statistical and machine learning. Currently, the package does not support this method. Generally, the procedure works as follows: Shuffle the data, split it into k groups. Let each group be the hold-out group exactly once, train a model on the remaining groups and evaluate it on the hold-out group. Finally, the model can be evaluated over the k scores achieved.

Permutation

Permutation is a resampling technique which is very useful in many-sample cases. Currently, the package does not support this technique. To explain it, consider a paired t-Test. With paired data \(X_i = \{X_{i1}, X_{i2}\}\) we randomly permute these values and conduct the paired t-Test in the bootstrap iteration. The evaluation works in the same way as for any other bootstrap procedure.
